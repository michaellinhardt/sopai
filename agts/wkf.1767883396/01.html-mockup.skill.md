# Skill: HTML Mockup Generator

## Purpose
Rapid UI/UX mockup generation using Bootstrap 5. Focus on structure and visual layout, not functionality.

## Principles
- **Minimal code** - Essential structure only, no bloat
- **UI/UX focus** - No backend logic, no JS unless essential
- **Mobile-first** - Responsive by default
- **Bootstrap defaults** - Use default theme unless user specifies colors

## Boilerplate

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1">
  <title>Mockup</title>
  <link href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.3/dist/css/bootstrap.min.css" rel="stylesheet">
</head>
<body>

<!-- CONTENT HERE -->

<script src="https://cdn.jsdelivr.net/npm/bootstrap@5.3.3/dist/js/bootstrap.bundle.min.js"></script>
</body>
</html>
```

## Workflow

1. **Clarify** - Understand what user wants (page type, sections, components)
2. **Structure** - Plan layout using Bootstrap grid
3. **Build** - Write minimal HTML with Bootstrap classes
4. **Deliver** - Single file, ready to open in browser

## Guidelines

| Do | Avoid |
|---|---|
| Use semantic HTML | Custom CSS unless essential |
| Bootstrap utility classes | JavaScript functionality |
| Placeholder content (`Lorem ipsum`, `#`) | Backend integration |
| Standard Bootstrap components | Over-engineering |

## Common Components

- **Navbar**: `navbar`, `navbar-expand-lg`, `navbar-light bg-light`
- **Hero**: `container`, `py-5`, `text-center`
- **Cards**: `card`, `card-body`, `row row-cols-1 row-cols-md-3 g-4`
- **Footer**: `footer`, `mt-auto`, `py-3 bg-body-tertiary`
- **Forms**: `form-control`, `form-label`, `btn btn-primary`

## Output

Save mockups to location specified by user or workflow folder. Use descriptive filename: `[name].mockup.html`
