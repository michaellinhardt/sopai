## Popularity/Adoption Claims

1. [claude_code_30min_limit]: Claude Code requires human intervention every approximately 30 minutes maximum for autonomous operation.

## Technical Architecture Claims

4. [kubernetes_loop_pattern]: Ralph's architecture is directly inspired by Kubernetes control loop philosophy of continuously driving current state toward desired state.
5. [four_core_artifacts]: Ralph operates on four core artifacts: idea.md, specs directory, plan.md, and loop.sh execution.
6. [loop_code_simplicity]: The Ralph loop code is a simple while-true bash loop that pipes a prompt into Claude.
7. [idempotent_crash_recovery]: Ralph loops are idempotent by design, allowing crash recovery by simply restarting the loop which re-reads state from disk.
8. [live_spec_editing]: Ralph allows live editing of specifications and plan files while the loop is running, with changes picked up on the next iteration.
9. [file_based_state]: All Ralph state is stored in files (specs, plan, source, bugs, feedback) with no in-memory state persisting across iterations.
10. [single_shot_execution]: The entire Ralph prompt is given to the model at once rather than step-by-step with human verification.
