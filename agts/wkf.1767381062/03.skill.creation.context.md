## Key Elements to Include

### From Guide Research

1. **YAML Frontmatter Requirements**
   - `name`: max 64 chars, lowercase, letters/numbers/hyphens only
   - `description`: max 1024 chars, must include what AND when to use

2. **File Structure Options**
   - Single file: `skill-name/SKILL.md`
   - Multi-file with scripts, references, assets

3. **Storage Locations**
   - Personal: `~/.claude/skills/skill-name/`
   - Project: `.claude/skills/skill-name/`

4. **Naming Conventions**
   - Gerund form preferred (verb + -ing)
   - Lowercase with hyphens
   - Avoid vague/generic names

5. **Best Practices**
   - Concise is key
   - Progressive disclosure
   - Test with all models
   - Include concrete examples

## Skill Design

2. Explain the required file format
3. Provide templates and examples
4. Cover common patterns
